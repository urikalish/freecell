# FreeCell

A FreeCell card game built as a mobile-first PWA.

- **Stack:** TypeScript · Vite · Vanilla CSS (no frameworks)
- **Author:** Uri Kalish
- **License:** MIT

---

## Getting Started

```bash
yarn          # install dependencies
yarn dev      # start Vite dev server (opens browser automatically)
yarn build    # typecheck + production build → dist/
yarn preview  # serve the dist/ build locally
yarn full     # format + lint + build (run before committing)
```

## Architecture

### State

`GameState` (defined in `types.ts`) is the single source of truth:

| Field | Type | Description |
|---|---|---|
| `freeCells` | `(Card \| null)[]` | 4 free-cell slots |
| `foundations` | `Card[][]` | 4 foundation piles, ordered by `FOUNDATION_SUIT_ORDER` |
| `tableau` | `Card[][]` | 8 tableau columns |
| `difficulty` | `DifficultyLevel` | Chosen difficulty level for the current game (`Beginner` | `Easy` | `Medium` | `Hard` | `Expert`) |
| `dealId` | `number` | MS FreeCell deal id — either chosen at random from the curated list for the selected difficulty, or specified directly by the player |
| `moveCount` | `number` | Incremented on every `executeMove` call |
| `gameStartTime` | `number` | Unix timestamp (ms) set on the first move; used to compute `elapsedSeconds` |
| `elapsedSeconds` | `number` | Mutated in place by the timer interval |
| `history` | `HistoryEntry[]` | Full move history enabling unbounded undo |

Difficulty is persisted in `localStorage` under the key `freecell-difficulty`. On every new game request the **New Game dialog** lets the player either select a difficulty level from a dropdown or enter a specific deal number (1–32,000). Both paths share a single **Play** button. When starting by difficulty, the chosen level is saved to `localStorage` and pre-selected next time. When starting by deal number, the difficulty is derived automatically by looking the id up in the curated `ms-deals.json` lists (falling back to `Medium` for ids that appear in none of them).

State is **immutable across moves** — `executeMove` and `undoLastMove` both call `cloneState` and return a new object. `elapsedSeconds` and `gameStartTime` are the only fields mutated in-place (`gameStartTime` is set on the first move; `elapsedSeconds` is updated on each timer tick).

Deals are generated by `msFreeCellTableau` in `deck.ts` using the same linear congruential generator as the original Microsoft FreeCell (`seed = (seed * 214013 + 2531011) & 0x7fffffff`). When starting by difficulty, the deal id is chosen at random from a curated subset in `src/data/ms-deals.json` that matches the chosen difficulty. When starting by deal number, `createNewGameWithDealId` uses the supplied id directly and derives the difficulty via `getDifficultyForDealId`, which reverse-scans the curated lists.

### Rendering

`renderer.ts` exports pure render functions. `main.ts` calls `app.innerHTML = renderGame(state, selectedCardId, validTargets)` on every state change — a full synchronous re-render. There is no virtual DOM or diffing. Two additional exports render overlays on demand: `renderVictoryOverlay()`, and `renderDifficultyOverlay(currentDifficulty)` (renders the New Game dialog with a difficulty `<select>` and a deal number `<input>`).

After re-render, animation hooks run in `setTimeout(..., 0)` / `requestAnimationFrame` callbacks so they operate on the freshly painted DOM.

### Move Flow

```
pointerup → handleTap → tryMove
  1. Capture source DOMRects for all moving cards
  2. `clearSelection()` + `state = executeMove(state, move)`
  3. Capture source rects for safe foundation auto-moves; execute them inline
  4. `render()` — full re-render
  5. `startTimerIfNeeded()` — starts the timer on the first move
  6. `animateCardMove([...moveRects, ...autoMoveRects])` — FLIP for all moved cards
```

**FLIP animation** (`animateCardMove`): captures source positions before render, reads destination positions after render, then clones each card at its destination offset by `translate(dx, dy)` and transitions it to `translate(0, 0)`. The real cards are hidden during flight and revealed on completion.

### Tap Cycle

Tapping a card with exactly one valid move executes it immediately (single-move shortcut). Tapping a card with multiple valid moves selects it and computes `tapCycleTargets` (all valid `MoveCandidate`s). Tapping the same card again cycles through those targets automatically, executing each move in turn. Tapping a valid target directly moves there immediately.

## CSS Custom Properties

All sizing and colour tokens are declared in `variables.css`. Key groups:

| Prefix | Purpose |
|---|---|
| `--color-brass*` | Primary accent colour family (`brass`, `brass-light`, `brass-dark`) |
| `--color-suit-red` | Red card suit colour |
| `--color-suit-black` | Black card suit colour |
| `--color-parchment*` | Text / UI element colours |
| `--glow-gold` | Gold box-shadow glow |
| `--surface-bg` | Table surface background |
| `--card-bg*` | Card background gradients (default, `movable`, `selected`) |
| `--card-*` | Card dimensions (`--card-width`, `--card-radius`, `--card-overlap`) |
| `--img-size-suit-large` | Large suit icon size |
| `--font-*` | Responsive font sizes (all in `vw`) |
| `--animation-*` | Animation durations (`fast`, `normal`, `slow`) |

Two Google Fonts are loaded in `index.html`:
- **Libre Bodoni** (italic, wght 400–700) — titles, subtitles, card values, roman rank; base body font
- **Federo** — stat labels, stat values, buttons, overlays

## Animations

| Function | Trigger | Mechanism |
|---|---|---|
| `animateDeal` | New game | CSS `reveal` keyframe with JS-computed per-card delays; `isAnimating` blocks input |
| `animateCardMove` | Every move | FLIP: fixed-position clones fly from source to destination via CSS `transition` |
| `animateLand` | Card land (internal) | `.card-land` scale bounce keyframe |
| `animateButtonPress` | Action button tap | `.btn-pressed` CSS class, 200 ms |
| `animateVictory` | Game won | Spark particles burst from random positions in `#victory-fireworks` |

## Key Invariants

- **No test suite** — correctness is enforced by TypeScript strict mode and ESLint.
- `isGameWon` returns `true` when all four foundations contain exactly 13 cards (`foundations.every(f => f.length === 13)`).
- After every player move (`tryMove`) and after starting a new game (`newGame` / `newGameWithDealId`), `findSafeFoundationMoves` is called to automatically advance unambiguous cards to foundations. In new-game functions this goes through the `autoMoveToFoundations` helper; in `tryMove` it is called inline to also capture animation rects before re-render.
- `isAnimating` flag gates all input during the deal animation only; card-move animations do not block input.
- Foundation suit order is fixed (defined in `types.ts` as `FOUNDATION_SUIT_ORDER`, re-exported from `suits.ts`) and must match the order foundations are stored in `GameState.foundations`.
- Card IDs are derived as `${suit}-${rank}` and are stable across clones — safe to use as DOM `data-card-id` keys.

## Browser Support

Targets modern evergreen browsers. Tested on desktop Chrome / Firefox / Safari and mobile Chrome / Safari.

> **SVG assets** (`public/img/*.svg`) must not contain an XML declaration or external DOCTYPE — both can cause mobile Chrome to silently refuse to render the file as a CSS `background-image`.
